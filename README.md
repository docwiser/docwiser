## 🚀 flagship projects (the main characters)

### 📸 **[PhotoLens](https://github.com/docwiser/photolens)** `Kotlin`
An open-source, **fully accessible photo gallery** powered by on-device AI models. generates image descriptions, recognizes text (OCR), and answers questions about images. built on the LiteRTLM framework. 

*why it's fire:* accessibility-first design + privacy through on-device processing = chef's kiss.

### 🎥 **[TubeBoard](https://github.com/docwiser/tubeboard)** `TypeScript`
Gemini-powered transcription + quiz creation workflow from YouTube videos. turn passive watching into active learning (no fluff).

*the narrative:* automate tedious content processing with AI so you can focus on what matters.

### 💬 **[FlowCast](https://github.com/docwiser/flowcast)** 
Fully open-source, self-hosted WhatsApp automation platform. administrative controls, API/webhook access, bulk message scheduling, subscription systems, billing infrastructure. this is enterprise-grade, built with *intention*.

*the flex:* complete feature parity with commercial solutions, but open, transparent, and yours to own.

### 🎵 **[SoundKit](https://github.com/docwiser/soundkit)** `JavaScript`
Performant, cross-platform, accessible music-streaming application. because music deserves to be accessible to everyone.

*design philosophy:* accessibility baked in from day one, not bolted on later.

### 📧 **[Mail System](https://github.com/docwiser/mail)** `JavaScript`
Simple, socket-based mail sending system to automate workflows. lightweight, reliable, built for scale.

### 🎓 **[YTWorkflo](https://github.com/docwiser/ytworkflo)** `Python`
Fully typed and documented YouTube API powered by YT-DLP and FastAPI. developer experience matters.

### 🍕 **[TasteOfTown](https://github.com/docwiser/tasteoftown)** `Vue`
Food delivery platform. full-stack mastery on display.

## 🎯 the breakdown (piece-by-piece)

### **accessibility first** ♿
every project i touch gets the accessibility treatment. WCAG compliance isn't a buzzword for me—it's a baseline. screen readers, keyboard navigation, semantic HTML, inclusive design patterns. *all the time.* accessibility is the ultimate flex.

### **open-source energy** 🔓
i believe knowledge should be free, and code should be collaborative. most of my work lives on GitHub. contribute, fork, remix—let's build together. no proprietary nonsense. just pure, open-source vibes.

### **performance obsession** ⚡
milliseconds matter. user experience matters. i'm constantly optimizing, profiling, and pushing for that buttery-smooth performance. lazy loading, code splitting, efficient algorithms—it's non-negotiable.

### **developer experience (DX)** 👨‍💻
